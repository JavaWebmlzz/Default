## 📌设计基本思路
  本项目采用 模块化 + 增量式开发 的总体设计思路。系统根据业务逻辑划分为若干可独立开发、测试和部署的功能模块，包括用户认证模块、课程内容展示模块、在线互动模块、学生功能模块等。
  在保证核心功能优先上线的前提下，通过迭代方式逐步扩展系统能力。
  
  整体结构采用 **前后端分离** 的架构模式：前端负责界面展示与交互逻辑，后端负责业务处理、权限验证和数据管理；
  两者通过统一的 RESTful API 通信。数据库部分保证数据的一致性、可扩展性与安全性，从而实现高可维护的系统架构。

## 📌技术选型

层级：                 技术方案                     |                   选择理由  

前端： HTML5 + CSS3 + JavaScript（或 Vue / React）  | 技术成熟、生态丰富，适合快速构建响应式页面 

后端： Java Servlet / JSP                          | 简单稳定，适合课程项目；易于实现 Session、Cookie、权限控制

数据库： MySQL                                     | 关系型数据库，可靠性高，结构清晰

通信方式： RESTful API + JSON                      | 前后端分离标准方式，结构清晰、扩展性强  

部署环境： Tomcat 11.0.11                                  | 与 Servlet/JSP 高度兼容，配置简单

工具链： Git + GitHub                              | 管理团队协作、版本控制、开发流程

编辑器： Jetbrain IDEA 2025.2.2 + JDK 17                             | 编辑器基础设置

## 📌 开发进度安排

项目采用 三阶段增量开发模型，每个阶段都保证可运行、可展示的成果。

* 第一阶段 — 核心功能搭建

搭建项目基本目录结构

建立数据库（用户表 / 课程表）

实现用户注册、登录、登出

完成 Session 校验与基础权限管理

实现简单的课程内容展示页面

* 第二阶段 — 主要业务功能完善

完善课程展示（分页、筛选）

实现学生功能（收藏课程、记录学习进度）

添加教师或管理员后台管理模块

优化页面 UI（基本布局、样式统一）

接口功能进行联调测试

* 第三阶段 — 最终集成与优化

开发在线互动模块（如评论、留言或问答）

整体功能测试、性能优化、错误处理完善

编写用户文档 + 项目 README

演示版本部署到 Tomcat 服务器

最终整体打包与报告汇总
